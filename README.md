## `04` — Case Studies

> Real projects. Real problems. Real outcomes.

<br/>

### ◈ Intelligent Expense Intelligence Platform

**Stack:** Django · REST Framework · PostgreSQL · PDF Generation · Celery

**The Problem:**
A business team was tracking expenses manually across spreadsheets. Reconciliation took hours each week, reports were inconsistent, and there was no audit trail or automated categorization.

**What I Built:**
A full-stack expense tracking system with a structured REST API backend, automated PDF report generation (scheduled and on-demand), role-based access control, and a PostgreSQL schema designed for long-term financial querying.

**Architecture decisions:** Decoupled report generation via background task queue. Normalized relational schema to support multi-currency and multi-category filtering without query degradation.

**Impact:**
- Eliminated ~80% of manual reconciliation time
- Enabled automated monthly reporting with zero human intervention
- Provided a clean audit trail for every transaction

---

### ◈ Sentiment Intelligence Engine

**Stack:** Django · MongoDB · NLP Models · REST API · Data Pipeline

**The Problem:**
A client needed to understand customer feedback at scale — reviews, support tickets, and social mentions — without reading them individually. The data was unstructured, high-volume, and growing daily.

**What I Built:**
A web application that ingests raw text, runs it through an NLP sentiment classification pipeline, stores results in MongoDB (designed for unstructured, schema-flexible data), and surfaces aggregated sentiment trends via a REST API — consumable by dashboards or downstream systems.

**Architecture decisions:** Chose MongoDB deliberately over relational DB for schema flexibility across feedback sources. Modular NLP layer allows model swapping without touching API contracts.

**Impact:**
- Transformed unstructured noise into structured, queryable intelligence
- Enabled trend analysis across thousands of data points in seconds
- Built for horizontal scaling — ingestion pipeline decoupled from analysis layer

---

### ◈ Autonomous Smart Home Control System

**Stack:** IoT Sensors · Embedded C · MQTT · Automation Logic · Arduino

**The Problem:**
A home automation use-case requiring real-time sensor-to-action decision loops — with low latency, low power consumption, and reliable failover behavior.

**What I Built:**
An embedded automation system connecting environmental sensors (temperature, motion, light) to actuators via a decision engine programmed in C. MQTT protocol handled device communication. Logic layer processed sensor state and executed conditional actions autonomously.

**Architecture decisions:** Edge-processing model — decisions made locally, not cloud-dependent. This ensured millisecond response times and offline resilience.

**Impact:**
- Fully autonomous operation with no human trigger required
- Sub-100ms sensor-to-action response time
- Modular sensor/actuator architecture — extensible without core rewrites
